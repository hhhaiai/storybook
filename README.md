# 📚 AI Storybook Generator（本地绘本生成器）

基于 Grok / OpenAI 兼容 API 的本地绘本生成工具，面向家庭友好内容：孩子能看，家长也愿意读，可做儿童绘本、通用绘本、漫画/条漫、二次元漫剧分镜、写真绘本等。

- ✅ 自动生成中文故事（可调 4-60 页）
- ✅ 自动生成插画（软扁平 / 水彩 / 蜡笔 / 二次元 / 国风二次元 / 漫画 / 写真 / 电影感等 25 种风格）
- ✅ 通用绘本 / 通用文学绘本 / 通用图像小说 / 通用写真绘本（家庭友好方向）
- ✅ 标准化风格系统（固定角色设定 / 角色一致性 / 场景检测 / 情绪映射）
- ✅ Web 控制台管理（选择风格、模型、主题）
- ✅ 本地 HTML 翻页阅读器
- ✅ PDF 导出

---

## 🚀 功能特性

- 封面海报页（标题 + 插画）
- 左图右文（标准绘本结构）
- 8 个标准角色（警察 / 武警 / 特警 / 刑警 / 消防员 / 男孩 / 女孩 / 妈妈）
- 支持固定角色设定：漫画/漫剧可指定姓名、年龄、发型、服装、体型、配饰和颜色，生成时会强制每页保持一致
- 7 个儿童故事模板 + 4 个通用故事风格（疗愈 / 都市 / 文学留白 / 悬疑）
- 自动重试图片生成 + 纯色检测
- 批量生成支持
- 本地运行，无需外部前端框架

## 📦 安装

需要 Go 1.21+：

```bash
# 编译
./build.sh

# 或手动编译
go build -o storybook ./cmd/server
```

---

## ⚙️ 配置

复制 `.env.example` 为 `.env` 并填入 API 密钥：

```bash
cp .env.example .env
```

编辑 `.env`：

```bash
# API 配置（必填）
USER_BASE_URL=https://your-api-endpoint.com/v1
USER_API_KEY=your-api-key-here

# 模型配置
TEXT_MODEL=grok-4.20-0309-non-reasoning
IMAGE_MODEL=grok-imagine-image-lite

# 服务配置
PORT=8080
```

支持 OpenAI 兼容接口，并内置 OpenAI Chat Completions、OpenAI Responses、Claude Messages、Gemini 文本协议适配；预设 Claude/Gemini 模型会自动选择对应协议。图片模型优先使用 OpenAI 兼容的 `/images/generations` 网关（支持 URL 或 `b64_json` 返回）。

### 故事风格（22 种）

温暖成长 / 友情万岁 / 亲情暖心 / 冒险探索 / 寻宝之旅 / 紧急救援 / 警察故事 / 消防救援 / 悬疑推理 / 科幻想象 / 爆笑趣事 / 魔法奇缘 / 合家疗愈 / 都市人生 / 文学留白 / 通用悬疑 / ...

### 作品类型（14 种）

儿童绘本 / 儿童故事书 / 连环漫画 / 四格漫画 / 少年漫画 / 条漫 / 科普绘本 / 童话绘本 / 无字绘本 / 真人绘本 / 通用绘本 / 通用文学绘本 / 通用图像小说 / 通用写真绘本

通用作品会自动切换为更克制、文学化、电影感的策划和分镜提示词，避免低幼口吻；同时保持家庭友好、安全温和的内容边界。

### 固定角色设定（漫画/漫剧一致性）

控制台里可以填写“固定角色设定”。建议写得越具体越稳定：

```text
主角小林，12岁男孩，黑色短发，蓝色连帽衫，圆框眼镜，身材瘦小但精神。
伙伴阿月，12岁女孩，白色短发，红色围巾，黄色斜挎包。
所有页面必须保持同一发型、服装、年龄、体型、配饰和颜色。
```

稳定建议：

- 固定姓名、年龄、发型、服装、体型、标志性配饰、主色调。
- 漫画/漫剧不要每页换衣服；如果确实要换装，写成“第二套固定服装”。
- 当前已在故事规划、插画描述、封面和单页图片 prompt 中重复注入固定角色约束，尽量降低角色漂移；不同图片模型仍可能有随机性，稳定性取决于模型本身。

## 🧠 生成逻辑

1. 规划故事名称与主题（LLM）
2. 注入固定角色设定（如有）并翻译为英文（用于图片 prompt）
3. 生成故事文本（中文）
4. 为每页生成插画描述（英文）
5. 生成插画（自动重试 + 纯色检测）
6. 渲染 HTML 绘本

## ⚠️ 注意

- 图片生成依赖网络和 API 可用性
- 自动重试最多 6 次（封面 10 次）；图片并发默认 3，可通过 `IMAGE_WORKERS=1-8` 调整，稳定优先建议 2-3
- 纯色图片会自动检测并重新生成
- `.env` 包含 API 密钥，已被 `.gitignore` 排除
